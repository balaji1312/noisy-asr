
These are the scripts to run the ASR model for our group for the project for ECE M214A - Digital Speech Processing

Group Members: Natarajan Balaji Shankar, Khushbu Pahwa, Sarthak Tanwani, Vishwas Shetty

######################################################################################

Contents:

wrapper.m - file to generate the features from the given wav files 
wrapper_test.m - file to generate features for just test data
QCN_RASTALP.m - Matlab function file
applyNoise2.m - Matlab function file
run_214A_3.sh - shell script to perform data preparation and training of GMM-HMM model in Kaldi
run_214A_test.sh - shell script for just testing model on new data
spc_data_prep_1.sh - kaldi script file
auto3.sh - shell script to automatically run the whole model after features are generated by Matlab
auto_test.sh - shell script to run model on test data
mono_spc - folder contained our trained model

######################################################################################

Requirements:

Kaldi
Matlab

######################################################################################

Intended File locations:

wrapper.m - s5/databases
wrapper_test.m - s5/databases
QCN_RASTALP.m - s5/databases
applyNoise2.m - s5/databases
run_214A_3.sh - s5
run_214A_test.sh - s5
spc_data_prep_1.sh - s5/local
auto3.sh - s5
auto_test - s5
mono_spc - s5/exp
(Note: if model is trained from scratch, mono_spc will be overwritten)

######################################################################################

Setting Things up:

1. Store required train/test data stored in egs folder inside Kaldi. If a noisy/clean file list is present along with the data, it is recommended to store the files under s5/databases.
2. Add s5/databases to Matlab Path

######################################################################################

Instructions to train new model:

1. cd to directory where examples are stored within kaldi (s5)
2. Execute run_214A_3.sh to perform stage 0 (zsh run_214A_3.sh 0 0 0 1)
3. Modify wrapper.m to refer to generated train_spc/wav.scp file (modify path of variable $pioth)
4. Run wrapper.m once
5. Run auto3.sh (zsh auto3.sh)

######################################################################################

Instructions to test model on existing data:

1. Run wrapper_test.m twice, once for clean and noisy data
2. Copy mono_spc to s5/exp
3. Run auto_test.sh (zsh auto_test.sh)
